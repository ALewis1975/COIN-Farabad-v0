# Campaign Persistence Reset Coverage

## Scope
This inventory documents every top-level key declared by `functions/core/fn_stateInit.sqf`. These keys are persisted inside the single `ARC_state` blob in `missionProfileNamespace`.

## Lifecycle contract
- **Init:** `ARC_fnc_stateInit` declares the versioned default key and value.
- **Load:** `ARC_fnc_stateLoad` sanitizes persisted `ARC_state`, drops nil values, and merges persisted values over current defaults so newly added keys appear after restart.
- **Save:** `ARC_fnc_stateSave` writes the whole `ARC_state` pairs array back to `missionProfileNamespace` and calls `saveMissionProfileNamespace`.
- **Reset:** `ARC_fnc_resetAll` replaces `ARC_state` with current defaults, then applies reset-specific reseeds/rerolls and immediately saves/publishes. AIRBASE and CASREQ runtime/public mirrors are explicitly cleared there.

## Subsystem reset notes
- **AIRBASE v1:** `airbase_v1_queue`, `airbase_v1_records`, and related control keys persist across normal init/restart. Hard reset clears queue, records, clearance requests/history/events, runway locks, active execution, notification state, and publishes a clean AIR tab snapshot.
- **CASREQ v1:** Compact records persist for active/JIP workflows. Hard reset clears records, open/closed indexes, sequence, and the public CASREQ bundle/revision.
- **Unknown legacy keys:** `resetAll` now resets the authoritative blob from defaults, so keys not declared below are removed from future saves unless code re-adds them intentionally via `ARC_fnc_stateSet`.
