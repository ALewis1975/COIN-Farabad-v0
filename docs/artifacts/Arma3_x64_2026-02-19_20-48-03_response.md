## Issues found and fixed

### P1 — `lineSpacing` invalid in S2 Intel structured text (24 RPT warnings)
**Symptom:** 24× `Unknown attribute lineSpacing` in RPT, appearing whenever the Intel (S2) detail pane was rendered.  
**Root cause:** `fn_uiConsoleIntelPaint.sqf:1384` wrapped `_txt` in `<t lineSpacing='1.15'>…</t>` before passing to `ctrlSetStructuredText parseText`. `lineSpacing` is a config-class property for `RscStructuredText` controls, not a valid inline attribute for `parseText`/structured text markup.  
**Fix:** Removed the wrapper; `_txt` (already well-formed structured text) is passed directly to `parseText`.  
**File:** `functions/ui/fn_uiConsoleIntelPaint.sqf`

### P1 — `Land_FlatTV_01_F` absent from `ARC_consoleTerminalClasses` canonical default (tower-screen proximity broken)
**Symptom:** Tower-screen objects (`Land_FlatTV_01_F`, two confirmed in RPT lines 8188–8189) did not grant console access to players without a tablet, despite PR #265 claiming the fix.  
**Root cause:** PR #265 added `Land_FlatTV_01_F` to the hardcoded fallback inside `fn_uiConsoleCanOpen.sqf` but did not add it to the canonical default in `data/ARC_ConfigData.sqf`. The function reads `ARC_consoleTerminalClasses` from missionNamespace (provisioned by ConfigData) first; the hardcoded fallback only fires when the variable is empty or wrong-type, which it is not (ConfigData sets a valid non-empty array). So the new classname was silently ignored.  
**Fix:** Added `"Land_FlatTV_01_F"` to `ARC_consoleTerminalClasses` in `data/ARC_ConfigData.sqf`.  
**File:** `data/ARC_ConfigData.sqf`

### P2 — Two objective toggles missing from known-consumer registry (startup WARN spam)
**Symptom:** Two `[ARC][CONFIG][WARN] Toggle '…' is declared in initServer.sqf but missing from the known-consumer registry.` lines at every server startup (lines 8025–8026, 16241–16242).  
**Root cause:** `ARC_objectiveScaffoldEnabled` and `ARC_objectiveMeetUseAI` were added to `_arcDeclaredServerToggles` but never added to `_arcKnownToggleConsumers`. Both are "declared for future feature; currently not consumed" per their inline comments.  
**Fix:** Added both to `_arcKnownToggleConsumers` with descriptive "declared for future feature; not yet consumed" notes.  
**File:** `initServer.sqf`

### P2 — CIVSUB `BACKGROUND_CHECK` `_ensureFn` forEach unguarded throw path
**Symptom:** RPT line 16484 — `[CIVSUB][ERR] BACKGROUND_CHECK action script error civ=Abdul-Aziz Gailani actor=76561198027320796 step=IDENTITY_TOUCH did=D14`. Error was caught at dispatcher level; player received "Background Check failed — try again" (fail-soft working).  
**Root cause:** The `_ensureFn` call inside the deps-check `forEach` loop was unguarded. If `compile preprocessFileLineNumbers` threw (e.g., corrupted file, missing file, or transient preprocess error), the throw propagated through `forEach` and escaped the background-check function, triggering the dispatcher-level `isNil` catch.  
**Fix:** Wrapped `_ensureFn` call in `private _nilDep = isNil { _fn = … call _ensureFn; }` so any compile/resolve error is absorbed within the iteration and causes only `_identityDepsOk = false` (returning INCONCLUSIVE) rather than an uncaught throw.  
**File:** `functions/civsub/fn_civsubContactActionBackgroundCheck.sqf`
